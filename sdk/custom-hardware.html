Custom hardware

Hardware requirements

The core SDK can be integrated on any device with adequate sensors and processing capabilities. The minimum requirement is one camera and a consumer-grade MEMS IMU. The recommended sensor configuration depends on the use case. For example, inside-out tracking should use a stereo or multi-camera fisheye global shutter camera setup, and aerial use cases greatly benefit from a barometer sensor.

As a rule of thumb, CPU resources equivalent to approximately one ARM Cortex A72 core (e.g., one core in Raspberry Pi 4) are required. The exact requirements depend on a number of factors such as the number of cameras, resolution, frame rate, and the available computer vision accelerators in the platform.

It is also crucial that the camera and IMU are properly synchronized and that the IMU registers are correctly configured (see our Calibration Manual for more details). This requires low-level hardware support and generally does not work out-of-the-box, if the IMU data originates from an external device such as a flight controller or IMU dev kit.

Contact us for more information.

Input API

Custom hardware can is supported through the Input API where the camera images (or accelerated features and descriptors) and IMU data are manually fed into the SDK. Camera calibration also needs to be manually handled.

The Input API is available with a commercial license. Contact us for more information.

The publicly available Spectacular AI SDK versions are free for non-commercial use.
